# 1c-batch

Навык ([Agent Skill](https://agentskills.io)) для AI-агентов — кроссплатформенная автоматизация пакетных операций с платформой 1С:Предприятие 8. 
Python-порт проекта [1c-batch](https://github.com/vladimir-kharin/1c-batch) с расширенным набором команд (40+).

<details>
<summary>Что такое Agent Skills?</summary>

Открытый стандарт расширения AI-агентов (Claude Code, Cursor и др.). Навык загружается в контекст по требованию: агент видит только имя и описание (~100 токенов), а полные инструкции подключает когда они нужны.

</details>

## Что умеет

| Группа | Операции |
|---|---|
| Запуск | Предприятие, конфигуратор |
| Конфигурация | Выгрузка/загрузка XML (полная/инкрементальная/частичная), .cf файлы, обновление БД |
| Расширения | Выгрузка/загрузка XML, .cfe файлы, удаление, список |
| Обработки | Сборка/разборка EPF/ERF в XML |
| База данных | Выгрузка/загрузка .dt, создание ИБ, тестирование и исправление |
| Проверки | Проверка конфигурации и модулей |
| Сравнение | Объединение и сравнение конфигураций |
| Свойства | Выгрузка/загрузка свойств (модули, макеты, права) |
| Обслуживание | Сокращение журнала регистрации |

## Установка

1. Скопируйте папку `src/1c-batch` в каталог навыков вашего проекта. Агент с поддержкой Agent Skills обнаружит навык автоматически.

   Например, для Claude Code — в `.claude/skills/`.

2. Установите CLI-инструмент:

   В каталоге 1c-batch/scripts :

   ```bash
   pip install .
   ```

## Настройка

Создайте в **корне проекта** файл `.1c-devbase.json` по образцу `assets/.1c-devbase.json.example`:

```json
{
  "platform_path": "/opt/1cv8/x86_64/8.3.27.1786/1cv8",
  "connection": {
    "type": "file",
    "path": "/home/user/Documents/MyBase"
  },
  "user": "Администратор",
  "password": "",
  "cf_dir": "src/cf",
  "cfe_dir": "src/cfe",
  "log_dir": "logs"
}
```

- Если `platform_path` не указан, платформа определяется автоматически (Linux, macOS, Windows).
- Для серверной базы: `"type": "server"`, `"server": "server-1c"`, `"base": "erp_dev"`.

> Добавьте `.1c-devbase.json` в `.gitignore` — файл содержит учётные данные.
